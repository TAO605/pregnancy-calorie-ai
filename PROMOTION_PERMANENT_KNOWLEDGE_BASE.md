# 推广前期准备永久知识库

## 工具访问类问题

### 第三方 SEO 工具跳转登录页

- 适用场景：访问 Semrush 镜像、SEO 数据后台、第三方订阅工具时被跳转到登录页。
- 问题表现：目标 URL 自动跳转到 `/login`，页面显示用户名、密码、登录按钮。
- 风险点：登录需要用户个人账号信息或第三方平台权限，Agent 不应索取、输入或保存密码。
- 解决方案：
  1. 停止自动化数据采集。
  2. 记录问题日志，标注关键词、搜索量、KD 等数据为“数据不可得”。
  3. 请用户在自己的 Chrome 中手动登录目标平台。
  4. 用户确认已登录并能进入工具后台后，再继续执行。
- 后续执行规则：
  - 未登录成功前，不得生成任何关键词表、搜索量、KD 或竞争数据。
  - 即使有种子关键词，也不得自行扩写成“真实数据表”。

### Chrome CDP 未连接导致无法访问登录态或交互型工具

- 适用场景：需要操作第三方 SEO 工具、Semrush 镜像、登录后后台、动态筛选页面。
- 问题表现：`web-access` 前置检查显示 `chrome: not connected`。
- 原因：Chrome 没有开启远程调试授权，Agent 无法连接用户日常 Chrome 环境。
- 解决方案：
  1. 打开 Chrome。
  2. 在地址栏输入 `chrome://inspect/#remote-debugging`。
  3. 勾选 `Allow remote debugging for this browser instance`。
  4. 保持 Chrome 打开。
  5. 回到 Codex 告知可以继续。
- 后续执行规则：
  - 未恢复 CDP 前，不得尝试生成关键词搜索量、KD、DR 或竞争数据。
  - 若真实工具数据不可访问，输出必须标注“数据不可得”。
  - 不得使用估算值、行业均值或记忆数据替代真实工具查询。

## 数据真实性规则

### SEO 指标必须来自真实工具

- 适用数据：关键词搜索量、KD、DR、外链根域名数量、流量占比、竞争对手指标。
- 允许状态：真实工具查询结果；或明确标注“数据不可得”。
- 禁止状态：编造、估算、猜测、用未验证公开文章替代工具数据。

## 内容边界规则

### 不生成博客文章相关内容

- 禁止内容：博客文章标题、大纲、Meta 描述、写作指引、正文片段。
- 允许内容：非博客的工具数据整理、合规日志、操作步骤、推广执行流程、数据表结构。

### Semrush Keyword Magic Tool 中 KD “不可用”的处理规则
- 适用场景：第三方 Semrush 镜像 Keyword Magic Tool 返回关键词搜索量，但 KD、意图或结果数显示“不可用”。
- 风险点：不能把“不可用”解释成低竞争，也不能用其他关键词、行业均值或经验值补齐 KD。
- 正确处理：
  1. 保留原始关键词和搜索量，并把 KD 标为“数据不可得”。
  2. 排序和黄金关键词筛选只使用真实数字 KD。
  3. 若用户要求 KD≤30、KD≤20 等筛选，KD 不可得的关键词必须排除在有效名单之外。
  4. 如果有效关键词数量不足，记录真实数量和原因，不自动编造扩展表。
- 后续扩展方案：只能继续用真实工具查询更多修饰词或刷新工具指标；刷新指标可能消耗平台额度，执行前应谨慎确认。

### 竞争对手分析中 DR 不可得的处理规则
- 适用场景：用户要求 DR、根域名、外链质量，但当前可访问工具只提供 Semrush Authority Score。
- 风险点：Semrush Authority Score 不是 Ahrefs DR，不能混用或改名。
- 正确处理：
  1. DR 字段写“数据不可得”。
  2. 另列 Semrush Authority Score、引荐域名、反向链接作为可验证替代指标。
  3. 自然流量、付费流量、自然关键词必须按工具真实显示填写；不可用就写“不可用”。
  4. 不使用公开文章或经验值补 DR。

### Google 搜索本地超时的备用流程
- 适用场景：Chrome CDP 打开 Google 搜索页超时或被网络环境阻断。
- 正确处理：
  1. 记录问题到问题日志。
  2. 使用可用联网搜索工具获取自然搜索结果。
  3. 后续指标仍回到真实 SEO 工具验证。
  4. 报告中说明搜索结果可能随地区、时间、个性化变化。

### 外链资源 DR 核实额度不足的处理规则
- 适用场景：用户要求 50+ 条 DR≥80 外链资源，但当前免费 DR 工具只能核实少量域名。
- 风险点：不能把平台知名度、Semrush Authority Score、DA 或个人经验当作 Ahrefs DR。
- 正确处理：
  1. 只给真实工具返回的域名填写具体 DR。
  2. 未核实域名统一写“数据不可得”或“待核实”。
  3. 仍可整理候选资源和操作步骤，但必须区分“已核实 DR≥80”和“候选待核实”。
  4. 记录工具额度限制和后续所需用户操作。
- 已验证免费工具限制：dachecker.io Bulk Ahrefs DR Checker 未登录免费额度为每次 3 个域名、总共 9 个域名。

### 海外母婴社区软植入回复规则
- 适用场景：Quora、Reddit、BabyCenter、What to Expect 等社区推广。
- 核心原则：
  1. 先回答问题，再提及工具。
  2. 链接必须和原问题直接相关。
  3. 如果与工具有关联，必须披露关系。
  4. 不重复复制同一模板，不刷屏。
  5. 不提供诊断、治疗、用药或替代医生建议。
  6. 对 gestational diabetes、hypertension、eating disorder history、twins/multiples、rapid weight change 等情况，引导咨询专业医护人员。
- 禁止话术：guaranteed、cure、diagnose、must、you are doing it wrong、better than a doctor。
