# Experience Library Usage Guide

Use this guide before starting any future overseas promotion task for AI Pregnancy Calorie Calculator.

## When to Use

Use the experience library before:

- New keyword research.
- Competitor analysis.
- Backlink building.
- Community promotion.
- Google Search Console indexing work.
- Google Analytics tracking review.
- Any SEO or growth task involving metrics.

## Files to Read First

1. `PROMOTION_MASTER_ISSUE_LOG.md`
2. `PROMOTION_PERMANENT_EXPERIENCE_LIBRARY.md`
3. `PROMOTION_ISSUE_LOG.md`
4. `PROMOTION_PERMANENT_KNOWLEDGE_BASE.md`
5. The most recent relevant batch file, such as:
   - `seo-keyword-research-batch1.md`
   - `COMPETITOR_ANALYSIS_REPORT.md`
   - `BACKLINK_DR80_BUILDING_LIST.md`
   - `COMMUNITY_REPLY_TEMPLATES.md`
   - `GA_GSC_TRACKING_GUIDE.md`

## Reusable Prompt Prefix

Use this at the start of future requests:

`Please first read PROMOTION_MASTER_ISSUE_LOG.md and PROMOTION_PERMANENT_EXPERIENCE_LIBRARY.md, then execute this task without repeating past mistakes. Do not fabricate SEO metrics. Mark unavailable data as data unavailable. Do not generate blog article content.`

## Decision Rules

### If a metric is unavailable

Write:

`数据不可得`

Do not estimate or replace it with another metric.

### If a tool requires login

Stop and ask the user to log in manually. Do not request or store passwords.

### If a community has strict promotion rules

Post helpful no-link replies first. Add a link only when the question is directly relevant and disclose affiliation.

### If a task asks for blog-related content

Stop and clarify that blog content creation is outside the allowed scope.

### If Google or SEO tool UI changes

Use the stable concept, record the changed path, and update the issue log.

## Weekly Review Checklist

Every Monday:

- Update GSC clicks, impressions, CTR, position, indexed pages.
- Update GA4 users, pricing views, premium clicks, checkout starts, subscriptions.
- Update backlink progress table.
- Review community replies and moderation status.
- Add new problems to `PROMOTION_MASTER_ISSUE_LOG.md`.
- Add reusable solutions to `PROMOTION_PERMANENT_EXPERIENCE_LIBRARY.md`.

## Output Standard

Every future promotion task should produce:

- A concrete deliverable file.
- A task issue log or update to the master issue log.
- A lessons learned update.
- A brief final summary with file links.
